# 운영규약

## 목적
- 프로젝트 협업 규칙과 상태 전이 규칙을 명시한다.

## 티켓 관리
- 필수 필드: `Status`, `Next Action`, `Lease`, `Evidence Links`
- 기본 전이: `Backlog -> Ready -> InProgress -> Review -> Done`
- 예외 전이: `* -> Blocked`, `Blocked -> Ready|InProgress`

## 글로벌 정책
- 삭제 금지: 문서는 삭제하지 않고 `DEPRECATED` 표기 후 아카이브한다.
- `Done` 전이는 Reviewer 전용이다.

## 운영 로그
- 변경 사유
- 결정 사항
- 후속 액션
